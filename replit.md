# Plan-It

A multi-tenant content preview and approval platform for social media agencies — agencies upload draft posts for clients to review, comment, and approve before publishing.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/plan-it run dev` — run the React frontend (port 25426)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string (already provisioned)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5 + Clerk auth (`@clerk/express`)
- DB: PostgreSQL + Drizzle ORM
- Frontend: React 19 + Vite + shadcn/ui + Tailwind CSS + wouter routing
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Auth: Clerk (Replit-managed) with proxy middleware
- Storage: Replit Object Storage (Google Cloud Storage backend)
- Build: esbuild (CJS bundle)

## Architecture decisions

- **Contract-first API**: OpenAPI spec → Orval codegen → typed React Query hooks + Zod validators. Never hand-write fetch calls.
- **Clerk proxy**: `clerkProxyMiddleware` sits at `/api/__clerk` so Clerk works on custom domains in production without CNAME setup.
- **Object storage presigned URLs**: Client POSTs metadata to `/api/storage/uploads/request-url` → gets presigned PUT URL → uploads file directly to GCS, bypassing the API server.
- **Platform icons**: `react-icons/si` v5 dropped `SiLinkedin` — use `artifacts/plan-it/src/components/platform-icon.tsx` which has an inline SVG fallback for LinkedIn.
- **Share link tokens**: UUIDs auto-generated by Drizzle `$defaultFn`, no external token library needed.

## Product

- **Agency dashboard**: total clients, pending/approved counts, recent activity feed
- **Client management**: grid of clients, per-client social account list, stats by platform/status
- **Post feed**: platform-styled post cards (square for IG/TikTok, 16:10 for LinkedIn), inline comments, one-click approve/request-changes
- **Upload flow**: presigned URL → direct GCS upload → post creation with media
- **Share links**: shareable preview URL for clients to review content without logging in
- **Roles**: TEAM_ADMIN can do everything; TEAM_EDITOR can create posts; CLIENT_APPROVER can only approve/reject

## Gotchas

- **After adding new DB tables**: run `pnpm run typecheck:libs` to rebuild `@workspace/db` declarations before running API server typecheck.
- **After changing OpenAPI spec**: run `pnpm --filter @workspace/api-spec run codegen` (includes `typecheck:libs` automatically).
- **`SiLinkedin` is missing from react-icons v5** — always use `getPlatformIcon()` from `platform-icon.tsx` for platform icons.
- **Orval generates `<OperationId>Params` for query params** — if an operation already has a path param named the same way, it collides. Remove query params or rename the operation.
- **API server is an esbuild CJS bundle** — changes need the workflow to rebuild (`pnpm run build` in the artifact) before they're live.
